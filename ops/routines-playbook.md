## 3. Schedule routines — `/schedule` in Claude Code

Three remote agents that run on cron, write artifacts to disk, and never tax your foreground context unless you read the output. See `.claude/context-design.md` § Routines for the architecture.

> **Don't double-schedule:** the existing daily meeting extraction at 8 PM IST runs via system cron (see `.claude/commands/daily-meeting-extract.md`). The routines below don't overlap with it.

### 3a. Morning brief — weekdays 8:00 AM IST

Pre-cooks your context. Output: `ops/daily/<YYYY-MM-DD>.md`.

Run in Claude Code:
```
/schedule create
```
Then provide:
- **Name:** morning-brief
- **Cron:** `0 8 * * 1-5` (08:00 Mon–Fri; adjust timezone in your scheduler)
- **Prompt:**
  ```
  You are the morning brief routine for Lalit's team-os.

  Today is {{date}}. Produce ops/daily/{{date}}.md with these sections:

  1. ## Overnight Jira changes
     Use mcp__claude_ai_Atlassian__searchJiraIssuesUsingJql with
     `(watcher = currentUser() OR assignee = currentUser() OR reporter = currentUser())
     AND updated >= -16h`. One bullet per issue: key, summary, status change.

  2. ## Slack mentions
     Search Slack with mcp__claude_ai_Slack__slack_search_public_and_private
     for "@lalit" and "Lalit" in the last 16h. One bullet per thread: channel, who, snippet.

  3. ## Today's calendar
     Use mcp__claude_ai_Google_Calendar__list_events for today. One bullet per event:
     time, title, attendees.

  4. ## Open refinement items
     Read product/roadmap/summer-2026/backlog-rank.md if present; list top 3.

  Never write to Jira/Aha. End-of-file: a single line "Generated by morning-brief routine".
  ```

### 3b. Day-end signal sweep — weekdays 7:00 PM IST

Catches what you missed during the day. Output: appends to `product/roadmap/<release>/signals/<feature>.md`.

Run:
```
/schedule create
```
Then:
- **Name:** day-end-sweep
- **Cron:** `0 19 * * 1-5`
- **Prompt:**
  ```
  Day-end signal sweep for Lalit's team-os. Today is {{date}}.

  For each feature listed in product/roadmap/summer-2026/CLAUDE.md (or any release-scoped CLAUDE.md):

  1. Search Slack with mcp__claude_ai_Slack__slack_search_public_and_private using the feature's keyword set (find them in product/roadmap/<release>/signals/<feature>.md "search-keywords" frontmatter if present).
  2. Search Gmail with mcp__claude_ai_Gmail__search_threads for the same keywords across the last 24h.
  3. For each hit, append a dated entry to product/roadmap/<release>/signals/<feature>.md with: source (slack/gmail), URL or thread ID, 1-line summary, who-said-it.

  Strictly append-only. Never modify existing entries. Never write to Jira/Aha.
  ```

### 3c. Monthly audit-context — 1st of month, 9:00 AM IST

Runs the `audit-context` skill. Output: `analytics/context-audit/<YYYY-MM-DD>.md`.

```
/schedule create
```
- **Name:** monthly-audit-context
- **Cron:** `0 9 1 * *`
- **Prompt:** `Run the /audit-context skill on the team-os repo. Today is {{date}}.`

### 3d. (Optional) Quarterly archive — 1st of Jan/Apr/Jul/Oct

Moves resolved signals to `_archive/` so live signal files stay short.

```
/schedule create
```
- **Name:** quarterly-archive
- **Cron:** `0 9 1 1,4,7,10 *`
- **Prompt:**
  ```
  Quarterly archive routine. Today is {{date}}.

  For each product/roadmap/*/signals/*.md whose feature's epic is closed
  (check via Jira `status = Done` for the linked epic) OR which has not been
  modified for >90 days: move to product/roadmap/*/signals/_archive/.

  Update each release's signals/INDEX.md after moves.
  ```
